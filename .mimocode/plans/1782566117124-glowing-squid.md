# Plan: Onboarding & Goals Setup

## Context
The app has an Onboarding component that's just an info carousel (no goal creation) and a GoalsManager with basic CRUD using a simplified Goal interface. The full Goal type already exists in `lib/types.ts` but isn't used by GoalsManager. Need to align everything and add the requested features.

## Changes

### 1. Rewrite `components/Onboarding.tsx`
- **Step 1**: Welcome screen — "What are your big goals?" with inline goal creation form (name, description, free-text category, target date, priority dropdown, target hours, weekly target)
- Users can add multiple goals before continuing
- Each added goal shows as a chip/card that can be edited or removed
- **Step 2**: Roadmap info — "You can create roadmaps later in Goals" (placeholder, no roadmap builder)
- "Skip for now" and "Continue" buttons
- On complete, saves all created goals to localStorage via `storage.ts`

### 2. Rewrite `components/GoalsManager.tsx`
- Use full `Goal` interface from `lib/types.ts` (via `storage.ts` CRUD functions)
- **Form fields**: name, description, free-text category, tag, target hours, weekly target, target date, priority (high/medium/low), color
- **Display**: show description, category badge, priority badge, status badge
- **Status logic**: auto-detect from progress (completed when targetHours reached, at-risk/behind based on pace) + manual override via dropdown (active/paused/completed/archived)
- **Operations**: Add, Edit, Pause/Resume (toggle active↔paused), Archive, Delete
- **Progress**: auto-calculated from sessions matching the goal's tag
- Remove old inline `Goal` and `Session` interfaces — use types from `lib/types.ts`

### 3. Files to modify
- `components/Onboarding.tsx` — full rewrite
- `components/GoalsManager.tsx` — full rewrite
- `lib/storage.ts` — may need minor tweaks for goal update helpers (updateGoal already exists)
- `lib/types.ts` — no changes needed (already has the full interface)

## Verification
1. `npm run build` — ensure no type errors

3. Create goals in onboarding → verify they appear in GoalsManager
4. Test CRUD: add, edit, pause/resume, archive, delete
5. Verify status auto-detection: log sessions with matching tags → progress updates
6. Test "Skip for now" in onboarding → goes to main app without goals
