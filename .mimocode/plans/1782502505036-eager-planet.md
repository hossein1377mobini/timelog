# COMPASS - Gap Analysis & Implementation Plan

## Current State Summary

The app is a **Next.js 16 (App Router)** single-page client-side app using **shadcn/ui + Tailwind v4**. All data persists in **localStorage**. It's a working time-tracking tool with goals, planning, and focus features.

---

2. **Task model is too simple** - Spec has: estimatedTime, scheduledDate, scheduledTime, tags, checklist, sessionId, pomodoroCount. Current has: text, done
3. **Evening reflection is flat** - Spec has structured lists. Current has single strings
4. **AnalyticsDashboard exists but is orphaned** - Not rendered anywhere
5. **Today/Week/Month header buttons are non-functional**

## Implementation Plan

### Phase 1: Data Model Enhancement (Foundation)
**Goal**: Enrich types to match spec without breaking existing data

1. **Enhance `Goal` interface** (`lib/types.ts`)
   - Add: `description`, `priority` (high/medium/low), `status` (active/paused/completed/archived)
   - Add migration: existing goals get default values

2. **Enhance `DailyTask` interface** (`lib/types.ts`)
   - Add: `estimatedTime` (minutes), `scheduledTime` (optional), `tags` ([]), `checklist` (ChecklistItem[]), `sessionId` (link to timer), `pomodoroCount`
   - Add `ChecklistItem` interface: `{ id, text, done }`

3. **Enhance `EveningReflection` interface** (`lib/types.ts`)
   - Change `accomplishments` from string to string[]
   - Change `distractions` from string to string[]
   - Add: `challenges` (string[]), `improvements` (string[]), `wins` (string[])

4. **Enhance `Interruption` interface** (`lib/types.ts`)
   - Add: `cause`, `recoveryTime`, `severity`

5. **Add `RoadmapNode` interface** (`lib/types.ts`)
   - `id`, `type` (phase/objective/task), `name`, `description`, `goalId`, `parentId`, `children[]`, `status`, `order`
